# 2026-05-16 Replace Cloth Range Issue

## 现象

在 `nudity -> cloth` 的 replacement 实验中，replace 已经开始起作用，但当前注入概念的作用范围偏大。

直接观察结果可以总结为：

- replacement 不是完全无效
- 它确实会把裸露语义往“cloth / covered / draped”方向推
- 但注入范围过宽，容易把整个人整体改写成“衣服/布料的质感”
- 最终不像“局部覆盖裸露区域”，更像“整个人被统一材质化”

## 当前判断

这说明当前问题已经不只是“有没有效果”，而是：

1. `replace` 有效果
2. `cloth` 概念过于泛化
3. 注入作用在整个人体结构上，而不是只作用于需要覆盖的区域

所以当前 replacement 更接近：

```text
nudity removal + global cloth texture rewrite
```

而不是理想中的：

```text
targeted body-region covering
```

## 含义

这条观察很重要，因为它说明：

- 继续单纯调大/调小强度不是主线解法
- 现在的瓶颈更像 replacement concept 的粒度和作用区域

也就是说，后续优化重点应从“强度”转到：

- 更窄的替换概念
  - 例如 `covered_torso`
  - 例如 `covered_chest`
  - 例如 `draped_upper_body`
- 或让注入只在目标概念高激活区域生效
- 或按 `nudity` 子概念分别配替换概念，而不是统一用 `cloth`

## 当前结论

当前 `replace` 路线值得继续保留，因为：

- 它已经显示出比纯 ablation 更接近“替换”的趋势

但当前版本还不能认为是理想替换，因为：

- 注入范围过宽
- 人物整体容易被布料质感主导
- 画面主体结构虽然保留一部分，但语义材质被过度重写

## `ordinary_person` 补充观察

之后又测试了一条更温和的 replacement 概念：

- `nudity -> ordinary_person`

目的不是注入布料纹理，而是把图像往“正常穿衣的人”这个更宽的语义先验上拉回去。

### 结果

这条路线也不是无效，而且比 `cloth` 更干净一些：

- 能看出“穿上了衣服/回到正常人物状态”的趋势
- 比 `cloth` 少一些全身布料材质化的问题

但它的问题也很明确：

- 替换效果不够稳定
- 更像“人物整体被重新拉回一个普通人轮廓”
- 而不是“局部给裸露区域补上衣物”
- 注入稍微增强后，容易影响头脸、肩颈和整体人体比例

### 参数现象

一版较弱注入时，注入分支几乎被擦除分支淹没，replace 效果不明显。

之后提高到约 `10%` 注入比例后，衣物/覆盖趋势更明显，但开始出现：

- 人脸偏大
- 头部轮廓不自然
- 上半身结构被重写

这说明：

- `ordinary_person` 比 `cloth` 更适合作为拓展方向
- 但它仍然太宽，容易回拉整个人物结构，而不是只修正目标部位

## 最终定位

当前阶段更合理的结论是：

1. 主线仍然是 `erasure`
2. `replacement` 已证明“可以做”
3. 但 `replacement` 目前更适合作为扩展结果，而不是主论文主结论

换句话说：

- 擦除已经相对稳定，可作为主结果
- 替换具备可行性，但当前副作用更大、局部性不足
- 论文表述上更适合放在扩展实验、讨论或附录中

## 下一步建议

优先级建议：

1. 主线继续以 `erasure` 为主
2. `replace` 作为扩展方向保留
3. 不再把 `cloth` 或 `ordinary_person` 当作最终替换概念
4. 后续如果继续做 replacement，优先改成更窄的概念
5. 例如 `covered_torso / covered_chest / clothed_upper_body`
6. 后续再考虑局部注入，而不是全图注入
