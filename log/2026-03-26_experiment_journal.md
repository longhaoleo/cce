# 实验记录：概念定位与擦除流程清理

日期：2026-03-26  
范围：`exp53 / exp54 / exp55 / batch`

## 1. 背景目标

- 建立“概念定位 -> 概念擦除 -> 结果对比”的可复现流程。
- 支持多概念批量 prompt 测试，并输出多时间窗干预结果。
- 统一概念字典路径，减少脚本间路径不一致导致的失败。

## 2. 主要调整过程

### 2.1 概念打分与对比

- 在 `exp53` 中引入 SAeUron 风格评分（基于负样本标准差惩罚）。
- 增加 TARIS 与 SAeUron 的可选并排导出，便于比较特征排名变化。

### 2.2 批量擦除流程

- 新增批量脚本 `scripts/run_batch_concept_erase.py`。
- 支持 `.txt/.csv` prompt 集合读取。
- 支持时间窗模式：
  - `single`：单窗口干预
  - `suite`：`custom + early/mid/late` 多窗口对比
- 复用共享 session，避免每个 case 重复加载 SDXL。

### 2.3 概念字典路径统一

- 输出目录从 `out_concept_dict_*` 迁移为 `concept_dict/<block_short>/<concept>/`。
- 代码读取逻辑优先新路径，并兼容旧路径兜底。
- 已存在字典目录完成迁移。

### 2.4 干预“平衡补偿”策略回滚

- 讨论后认为当前阶段优先保持干预机制简单可解释。
- 已移除/清理相关参数接口与无效兼容代码。
- 当前路径聚焦纯干预表达：`x ± scale * recon`。

## 3. 典型问题与经验

- `feature_rank_csv 不存在` 多数是 block 与概念字典目录不匹配。
- 模型路径尾部空格会导致加载失败。
- `OMP_NUM_THREADS` 非法值会触发 `libgomp` 报错。
- 批量脚本 `--prompts_path` 需要传文件路径，不是文件夹。

## 4. 当前结论

- 主流程已可稳定跑通：
  - `exp53` 产出概念字典
  - `exp54` 做时间窗干预对比
  - `run_batch_concept_erase.py` 批量导出效果
- 文档已重写为本项目版本，路径和参数与当前代码一致。

## 5. 下一步建议

- 固化一套“标准评测集 + 标准命令”作为每次改动的回归基线。
- 给 `log/` 建立按日期追加机制，保持每次实验最小记录。
