# 2026-04-28 Shared 擦除已有效，但对原图扰动较大

## 结论

当前 `SharedSAE` 路线下，`car` 概念擦除已经开始出现明确效果，不再只是“轻微扰动但概念基本还在”。

但与此同时，当前更有效的配置也明显带来了更大的原图扰动：

- 画面结构更容易发飘
- 稳定性下降
- 不是单纯“把 car 干净拿掉”，而是会较强地改动整体中间表示

所以当前阶段的判断是：

- **擦除方向已经打通**
- **但副作用仍然偏大**

后续优化重点不该再放在“能不能打进去”，而应该转向：

- 怎么减少对原图内容的破坏
- 怎么在擦除效果和图像稳定性之间找平衡

## 现象回顾

早期 Shared 擦除不理想时，主要问题是：

- `mid.0` 太弱
- `time_weight` 会把干预信号压得很薄
- 多层一起打但真实 `delta_over_x` 很小

后来切到更稳的 `v3` 配置后：

- 去掉 `mid.0`
- 关闭 `time_weight`
- 只保留 `down.2.1 / up.0.0 / up.0.1`

擦除效果明显增强，说明 Shared 路线本身不是无效，而是之前配置把干预压没了。

但增强后的代价也很明显：

- 干预量上来以后，对原图结构的影响变大
- `int_scale` 对观感非常敏感
- 即使把 `int_scale` 从 `20` 降到 `10` 或 `5`，仍然会感觉不够稳定

## 当前理解

这更像是进入了一个“高敏感区”：

- 不是 bug
- 也不是 `Shared` 完全没学会
- 而是现在干预已经足够强，开始真正改动扩散轨迹

一旦：

- 多个 block 同时干预
- 覆盖整个时间窗
- 不乘时间权重

那么即使 `int_scale` 数值看起来不大，实际有效扰动仍然可能很强。

## 当前默认判断

在目前可用配置里：

- `mid.0` 暂时不建议默认纳入
- `down.2.1 / up.0.0 / up.0.1` 是更可用的主干 block
- Shared 擦除已经“有效”
- 但还不能说“已经达到又干净又稳定的概念擦除”

## 后续调参重点

接下来优先关注：

1. 缩小干预覆盖范围，而不是只继续降 `int_scale`
2. 优先收窄时间窗
3. 必要时减少 block 数
4. 减少 `top_k`
5. 在保住擦除效果的前提下再追求稳定性

一句话总结：

**Shared 擦除已经不是“打不进去”，而是“打进去了，但现在有点太猛”。**
